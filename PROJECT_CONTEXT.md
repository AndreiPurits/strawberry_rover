# Контекст проекта Strawberry Rover

## Назначение проекта
Разрабатывается программная система для автономного ровера, который должен:
- двигаться между грядками,
- анализировать окружающую среду,
- распознавать ягоды,
- в дальнейшем управлять манипулятором для сбора урожая.

## Оборудование, которое уже есть
- Jetson AGX Orin
- USB RGB камера

Камера:
- уже подключена,
- поток изображения работает в ROS2,
- используется для разработки компьютерного зрения.

## Оборудование, которое планируется подключить

### LiDAR сенсор
- **Для чего нужен:** точное обнаружение препятствий и оценка свободного пространства.
- **Какие модули используют:** `rover_fake_lidar` (сейчас симуляция), будущая навигация и fusion-модули.
- **Что пока ограничено без железа:** финальная проверка точности обнаружения препятствий и работы в реальном поле.

### Стереокамера
- **Для чего нужна:** глубина сцены и более точное 3D-восприятие.
- **Какие модули используют:** будущие perception-модули и fusion-пайплайн.
- **Что пока ограничено без железа:** полноценная depth/disparity обработка на реальных данных.

### Манипулятор
- **Для чего нужен:** физический захват и сбор ягод.
- **Какие модули используют:** будущие модули управления манипулятором и task-планирования.
- **Что пока ограничено без железа:** отработка реального цикла подведения и сбора урожая.

### RTK GPS
- **Для чего нужен:** точное позиционирование на поле.
- **Какие модули используют:** навигация, планирование маршрутов, мониторинг.
- **Что пока ограничено без железа:** точная абсолютная локализация и геопривязка траекторий.

## Назначение симуляции лидара и виртуальной среды
Сейчас в проекте есть виртуальная среда ROS2/RViz, которая моделирует:
- движение ровера,
- сканирование лидара,
- коридор между грядками,
- грядки/клумбы и препятствия.

Почему это сделано:
- чтобы начать разработку навигации до поставки всего оборудования,
- чтобы тестировать движение и поведение ровера,
- чтобы заранее разрабатывать алгоритмы обнаружения препятствий,
- чтобы отладить движение между грядками,
- чтобы проверить целостность ROS2-архитектуры.

Ожидаемый результат этого этапа:
- готовые алгоритмы навигации,
- проверенная архитектура системы,
- подготовка ПО к подключению реального лидара.

Когда реальный LiDAR будет подключен, симулированный сенсор заменяется на реальный, а основная навигационная логика сохраняется. Такой подход заметно ускоряет разработку.

## Текущее состояние проекта (кратко)
- Архитектура ROS2 сформирована.
- Работает пакет системного запуска `rover_bringup`.
- Реализованы симуляция лидара и симуляция среды (коридор/грядки).
- Реализована симуляция движения ровера.
- Подключена реальная RGB-камера.
- Визуализация в RViz работает.
- Управление движением через `/cmd_vel` работает.
- TF-модель робота настроена и используется в системе.

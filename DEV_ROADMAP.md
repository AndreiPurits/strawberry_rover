# Дорожная карта разработки ПО Strawberry Rover

Документ описывает этапы разработки программной системы ровера для движения между грядками, распознавания объектов и последующего автономного сбора урожая.

## Этап 1 — Базовая архитектура ПО

**Цель этапа:**  
Подготовить надежный каркас ROS2-проекта, на который можно безопасно наращивать функциональность.

**Что разрабатывается:**  
- структура ROS2 workspace и пакетов  
- базовые правила взаимодействия между узлами  
- пакет запуска системы (`rover_bringup`)  
- базовая модель робота и TF-структура

**Ожидаемый результат:**  
Готовая архитектурная основа проекта с единым подходом к запуску и развитию модулей.

**Статус:** выполнено

---

## Этап 2 — Интеграция сенсоров

**Цель этапа:**  
Подключить реальные и виртуальные сенсоры в единую ROS2-среду.

**Что разрабатывается:**  
- подключение реальной RGB-камеры (`rover_perception`)  
- публикация `image_raw`, `camera_info`, debug-видеопотока  
- стабильный backend для камеры (V4L2, рабочие параметры устройства)  
- подготовка интерфейсов для дальнейшего расширения сенсорного стека

**Ожидаемый результат:**  
Стабильный поток сенсорных данных в ROS2, пригодный для алгоритмов компьютерного зрения и мониторинга.

**Статус:** выполнено (по RGB), в работе (расширение под новые сенсоры)

---

## Этап 3 — Среда симуляции

**Цель этапа:**  
Создать виртуальную среду, позволяющую разрабатывать навигацию до появления полного набора оборудования.

**Что разрабатывается:**  
- симуляция лидара (`rover_fake_lidar`)  
- виртуальная сцена: стены, грядки, коридор, маркеры RViz  
- симуляция движения ровера (`rover_simulation`) по `/cmd_vel`  
- связка симуляции с TF и визуализацией

**Ожидаемый результат:**  
Рабочий виртуальный полигон для тестирования движения, сканирования и логики управления.

**Статус:** выполнено

---

## Этап 4 — Навигация между грядками

**Цель этапа:**  
Реализовать устойчивое движение ровера внутри коридора между грядками.

**Что разрабатывается:**  
- узел навигации (`rover_navigation`)  
- управление скоростью и направлением по данным `/scan`  
- публикация `/cmd_vel`  
- debug-топики для анализа steering/centerline

**Ожидаемый результат:**  
Ровер уверенно движется по коридору в симуляции, алгоритм стабилен и повторяем.

**Статус:** в работе

---

## Этап 5 — Компьютерное зрение (поиск ягод)

**Цель этапа:**  
Научить систему выделять полезные объекты на изображении (ряд, растения, ягоды, препятствия).

**Что разрабатывается:**  
- предобработка изображения  
- первичные алгоритмы обнаружения объектов  
- публикация perception-результатов в ROS2  
- подготовка данных для навигации и будущего манипулятора

**Ожидаемый результат:**  
Базовая vision-система, выдающая полезные признаки для управления и анализа.

**Статус:** запланировано

---

## Этап 6 — Веб-интерфейс мониторинга системы

**Цель этапа:**  
Обеспечить удаленный мониторинг состояния робота с любого устройства.

**Что разрабатывается:**  
- веб-панель с визуализацией потоков и состояния системы  
- вывод видео, лидара, положения ровера и debug-информации  
- интеграция ROS2-телеметрии в backend/monitoring слой
- web control channel для ручного управления и переключения режимов
- lifecycle работы с маршрутами: построение, редактирование, исполнение

**Ожидаемый результат:**  
Интерфейс наблюдения и диагностики для команды разработки и заказчика.

### Подэтапы реализации Stage 6 (web navigation/control extension)

**6.1 Web control basics**
- start/stop команды из web UI
- manual/auto mode switching
- keyboard control
- joystick control
- backend contract для control-команд (без изменения sim/navigation на этом шаге)

**6.2 Route recording**
- route building mode во время движения ровера по грядкам
- запись траектории с RTK/geo привязкой
- привязка к online map
- сохранение метаданных: длина грядок, межрядье, snake-структура
- сохранение маршрута для последующего редактирования

**6.3 Route editing / map management**
- просмотр сохраненных маршрутов
- редактирование геометрии маршрутов
- spline-based коррекция траектории
- добавление/удаление рядов
- масштабирование route-структуры под фактическое поле

**6.4 Route execution**
- загрузка сохраненного маршрута
- отображение маршрута на карте
- запуск follow-route режима для ровера
- status/telemetry отображение прогресса выполнения маршрута

**Статус:** запланировано
